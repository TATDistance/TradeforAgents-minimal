# TradeforAgents-minimal

面向中国 A 股的 AI 交易分析与模拟工作台。

当前仓库已经不是“单只股票分析脚本”，而是一个带网页入口的组合系统：

- `8600`：AI 实时决策首页
- `研究中心`：自动选股、单股分析、候选卡片、分享页
- `复盘中心`：交易计划、模拟盘、复盘结果
- `8610`：调试面板

它适合：

- 中国 A 股 / ETF
- 个人使用
- 无券商 API
- 盘后研究、盘中观察、模拟交易、人工执行

它不适合：

- 全自动实盘下单
- 高频 / 秒级交易
- 券商账户自动同步

## 最新更新

### 第八阶段：一键闭环启动 + 类证券账户首页

这一版把系统从“能启动实时 AI 决策”继续推进成更像产品的闭环。

过去的两个现实问题是：

- 点击“一键启动 AI 实时决策”时，如果没有当天候选池，系统只能退回默认观察池
- 首页虽然已经有 AI 摘要和账户状态，但还不像证券账户首页，缺少监控池、图表和动作时间线

这一版主要补上了这两块。

#### 0. 最近补丁：watchlist 与图表稳定性修复

这一轮还补了两类容易让首页看起来“像坏了”的问题：

- 前导 `0` 的股票代码现在会稳定按字符串处理  
  例如 `002155 / 002353` 不会再因为 YAML 解析被错误变成 `1133 / 1259`
- 首页图表会优先选择有有效行情的标的  
  如果本地没有完整图表缓存，会先回退最近 quote / 最近历史日线；闭市后至少优先保证 K 线可回看

这意味着：

- 监控池下拉框里的股票代码和股票名会更稳定
- 图表不会因为选中了错误代码而空白
- 闭市后如果缺少完整分时点，仍会尽量显示最新价点或最近日线，而不是直接变成“什么都没有”

#### 1. 一键启动真正补齐了 watchlist 闭环

现在首页点击：

```text
一键启动 AI 实时决策
```

会按下面这条链执行：

```text
检查当前 watchlist
-> 若缺失 / 过期 / 仅有默认观察池，则先自动选股
-> 同步 runtime watchlist
-> 启动实时引擎
-> 启动 8610 调试面板
```

系统会明确记录并展示当前监控池来源：

- `auto_selector_today`
- `recent_candidates`
- `default_fallback`

这意味着：

- 用户不需要再先手动进入研究中心执行自动选股
- 实时引擎启动后一定有可用监控池
- 自动选股失败时，也会自动回退到最近候选池或默认观察池

#### 2. 首页升级成类证券账户前台

首页现在不再只是状态卡片，而是新增了真实可用的展示层：

- 当前监控池与持仓池
- 分时图
- K 线图
- 账户收益曲线
- 动作时间线
- 可执行机会摘要

也就是说，用户打开首页后，不只是知道“系统在跑没跑”，还可以直接看到：

- 今天系统盯哪些票
- 哪只票当前最值得关注
- AI 最近打算怎么做
- 账户和动作是怎么变化的

#### 3. 首页和 8610 的分工更清楚

- 首页负责回答：系统现在在做什么
- 8610 负责回答：系统为什么这么做

首页看起来更像“AI 实时交易前台”，8610 保持为“高级调试后台”。

### 第七阶段：从“固定轮询”升级到“事件驱动 AI 决策”

这一版的重点，不是再多加一个策略或页面，而是把整个实时决策主链升级成更像真实盘中系统的结构。

过去的主要问题是：

- 系统虽然叫实时，但更像“每隔几秒全量重算一次”
- 很多股票会进入观察，但长期很难真正转成可执行动作
- 用户经常只能看到“没有买入”，却不知道到底是股票不够强，还是当前阶段/风险不允许

这一版主要解决了这三个问题。

#### 1. 实时引擎升级为事件驱动

系统不再默认每轮对所有股票全量重算，而是优先按这些变化触发：

- 价格变化
- 特征变化
- 市场状态变化
- 持仓变化
- 账户状态变化

只有受影响的股票才会重新进入决策链。

这带来的直接效果是：

- 更接近真正的实时系统
- 减少无意义重复计算
- 降低“轮询拥堵但没有有效动作”的情况

#### 2. 单一综合分改成双分体系

现在不再只看一个模糊的 `final_score`。

系统会明确区分：

- `setup_score`
  - 表示这只股票值不值得继续观察
- `execution_score`
  - 表示这只股票当前是否值得立刻执行

这意味着：

- 一只股票可以“值得观察，但当前不该买”
- 系统也能明确告诉你，问题到底出在：
  - 当前阶段
  - 风险模式
  - 执行权限
  - 还是股票本身不够强

#### 3. AI 不再只是存在感很弱的附属分数

这一版里，`ai_score` 会真正参与执行判断：

- 当 AI 与策略方向一致、市场状态匹配、账户风险允许时，会提供正向加分
- 当市场高波动、风险模式偏防守、价格位置不理想时，会提供压制或负向影响

这样做的目标是：

- 让 AI 真正参与“增强确定性”
- 而不是长期停留在“有 AI，但结果几乎总是 0 分加成”

#### 4. 首页不再只是展示结果，而会解释原因

现在首页会更明确展示：

- 当前可建仓机会数
- 当前可减仓机会数
- 为什么现在没有动作
- 哪几只股票最接近执行阈值
- `setup_score / execution_score / ai_score / 风险惩罚 / 阶段惩罚`

所以用户不用再自己猜：

- 为什么现在没买
- 为什么这只票只是观察
- 为什么这只票已经接近可执行

#### 5. 报表和评估也跟着升级了

这次不只是首页和实时链路变了，评估层也新增了事件驱动相关统计：

- 触发次数
- 有效触发率
- 触发后真实成交率
- 平均 `setup_score`
- 平均 `execution_score`

这样你后面看日报和评估时，看到的不再只是“赚了还是亏了”，还会知道：

- 系统到底触发了多少次
- 其中有多少次真的转成动作
- 是不是仍然过于保守

#### 这一版适合谁

如果你希望这个项目更接近：

- 盘中实时 AI 多因子决策模拟系统
- 而不是只做盘后候选和次日计划

那这一版就是朝这个方向迈出的关键一步。

## 第七阶段：事件驱动与双分体系

当前实时引擎已经不再只靠“固定 10 秒全量轮询所有股票”做决策。

新版本支持：

- `event_driven_mode`
- `polling_mode`

默认建议使用：

- `event_driven_mode`

它的运行逻辑是：

```text
行情更新 / 特征变化 / 市场状态变化 / 持仓变化 / 账户变化
-> 触发事件
-> 只重算受影响股票
-> AI 决策引擎
-> 风控
-> 模拟执行
```

这样做的目的有两个：

1. 避免每轮对所有股票全量重算
2. 解决“系统过于保守、长期只有观察、几乎没有动作”的问题

### setup_score 与 execution_score

当前首页和 8610 都会区分两层分数：

- `setup_score`
  - 表示这只股票值不值得继续盯
  - 不受午休、收盘后这类交易阶段限制直接压制

- `execution_score`
  - 表示这只股票当前是否值得立刻执行
  - 会受到交易阶段、execution gate、账户状态、风险惩罚影响

所以现在看到“值得观察但不买”，通常表示：

- `setup_score` 还不错
- 但 `execution_score` 还没过执行阈值

### 当前默认阈值

```yaml
scoring:
  min_setup_score_to_watch: 0.35
  min_execution_score_to_buy: 0.55
  min_execution_score_to_reduce: 0.45
```

这比旧版本“一个 0.65 阈值控所有场景”更合理：

- 观察条件和执行条件分开
- 建仓和减仓灵敏度分开
- 更容易真正产生自主交易动作

## 第八阶段：监控池生命周期与首页图表化

第八阶段是在第七阶段基础上的产品层升级，重点不是改引擎，而是补齐真实使用闭环。

### watchlist 生命周期

系统现在会把当前监控池当成一个有生命周期的对象管理：

```json
{
  "symbols": ["300750", "688525"],
  "source": "auto_selector_today",
  "generated_at": "2026-04-01T09:31:22",
  "valid_until": "2026-04-01T15:00:00",
  "trading_day": "2026-04-01"
}
```

优先级是：

1. 今日自动选股结果
2. 最近一次有效候选池
3. 默认观察池（最后兜底）

### 首页图表层

首页现在新增了真正可用的图表层：

- `当前监控池与持仓池`
- `分时图`
- `K 线图`
- `账户收益曲线`
- `动作时间线`

图表不是静态占位，而是直接读取：

- 盘中缓存的分时点
- 历史 K 线缓存
- 账户快照
- 最近动作记录

### 动作时间线

首页动作时间线会区分：

- `intent`
- `filled`
- `rejected`

所以现在能直接看出：

- 什么时候想买
- 什么时候真的成交
- 又是在哪一步被阶段或风控拦住

## 快速开始

### 方式一：网页入口

```bash
git clone https://github.com/TATDistance/TradeforAgents-minimal.git
cd TradeforAgents-minimal
bash start.sh web
```

然后打开：

```text
http://127.0.0.1:8600/
```

推荐使用顺序：

1. 先在 AI 首页填写 `DeepSeek API Key`
2. 直接点击 `一键启动 AI 实时决策`
3. 看首页的监控池、图表、账户和动作时间线

5. 想看底层输入和日志时，再打开 8610 调试面板

## 现在的运行逻辑

当前系统同时保留两条链路：

### 旧链路

```text
策略出候选信号
-> AI 审批
-> 风控
-> 模拟执行
```

### 新链路

```text
实时行情
-> 特征构建
-> AI 决策引擎
-> 风控
-> 模拟执行
```

并支持：

- `legacy_review_mode`
- `ai_decision_engine_mode`
- `compare_mode`

## A 股交易时段约束

当前系统已经受交易日历与交易阶段驱动：

- 非交易日：不新增真实模拟成交
- 午休：不成交
- 收盘后：只做分析、复盘、明日准备动作
- 上午 / 下午连续竞价：才允许真实模拟成交

首页和 8610 会显示：

- 今天是否交易日
- 当前处于哪个交易阶段
- 现在是否允许开仓
- 现在是否允许真实模拟成交

## 常见问题

### 1. 没在页面里填 API Key，为什么首页还有 AI？

因为当前 AI 可能来自以下任一来源：

- 首页已保存的浏览器本地配置
- 仓库根目录 `.env`
- 以前生成的 `decision.json` 研究缓存

首页现在会直接显示“AI 来源”，不要只看有没有动作。

### 2. 为什么首页会显示东财错误？

东财公开接口偶尔会有 DNS、网络或限流问题。系统现在会：

- 自动重试
- 尽量回退缓存 / 本地结果
- 不把旧错误一直挂成当前故障

### 3. 为什么现在不能成交？

先看首页里的：

- 交易日
- 当前阶段
- 允许成交

如果是非交易日、午休或收盘后，系统可以继续分析，但不会新增真实模拟成交。

## 当前限制

- 不接券商 API，不做实盘自动下单
- 东财是公开行情源，不等于机构专线
- `decision.json` 仍然是研究缓存的重要来源
- Streamlit 调试面板更偏工程视图，不是最终用户界面
